# uni-app 实战

##首页验证是否可以在线
1. 在onlunch里面判断是否在线（就是判断token是否存在）
2. token不存在直接跳转到登录页
3. token存在就去校验是否token过期，过期就去登录页
4. 如果token未过期，则在此页面并显示出名字
	

## 登录以及注册
* 账号密码登录
1. 登录传入的参数  username:长度大于3	 密码:长度大于6;	captcha验证码:str,scene:'login',deviceId（设备码）:str
2. 设备码是自己生成的一个查询登录机器的随机数，方法写在了utils文件下,并保存在'uni_deviceId'里面（操作系统的版本加拼接32位长的字符串）
2. 登录成功后，保存token到 uni_id_token 保存username到：username；
